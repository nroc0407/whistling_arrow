# 휘파람 음정 기반 지속형 화살 제어 기믹 명세

- 문서 버전: 1.0
- 기록일: 2026-07-15 (Asia/Seoul)
- 기준 구현: `whistling_arrow.html`
- 기준 리비전: `003046ee2367a1a183c6483eefe43356a88a0e4e`

## 1. 목적과 범위

이 문서는 기준 구현에 들어 있는 게임 제어 기믹을 엔진과 표현물로부터 분리하여 재현 가능한 형태로 정의합니다. 특허 청구항이나 추상적인 아이디어에 대한 독점 선언이 아니라, 실제 구현의 구조와 매개변수를 식별하기 위한 기술 기록입니다.

기믹의 한 문장 정의는 다음과 같습니다.

> 실시간 마이크 입력에서 추정한 기본 주파수를 귀환·중립·공격의 세 명령으로 분류하고, 하나의 지속형 발사체가 명령에 따라 홈 복귀·관성 부유·최근접 목표 자동추적을 수행하게 하는 게임 제어 방식.

## 2. 핵심 불변 조건

1. 플레이어가 제어하는 화살은 공격마다 새로 발사되는 탄환이 아니라 게임 공간에 계속 존재하는 단일 발사체입니다.
2. 플레이어의 음정은 화살의 좌표나 방향을 직접 지정하지 않고 `RETURN`, `FLOAT`, `ATTACK` 중 하나를 선택합니다.
3. `ATTACK` 상태의 목표 선택과 선회는 자동이며, 기준 구현에서는 매 갱신 시점의 최근접 생존 적을 사용합니다.
4. 무음, 신뢰도가 낮은 입력, 기준 음정 주변의 중립 구간은 `FLOAT`로 귀결됩니다.
5. 화살의 위치, 속도와 방향은 상태 전환 뒤에도 이어집니다. 따라서 같은 입력이라도 전환 직전의 운동 상태에 따라 궤적이 달라질 수 있습니다.

## 3. 입력 처리와 명령 분류

### 3.1 음정 추정

기준 구현은 마이크의 시간 영역 샘플을 다음 순서로 처리합니다.

1. 샘플 평균을 빼서 DC 오프셋을 제거합니다.
2. RMS를 계산하고 최소 진폭보다 작으면 유효 음정 없음으로 판정합니다.
3. 분석 주파수 범위에 해당하는 주기 후보마다 정규화 자기상관을 계산합니다.
4. 가장 큰 상관값이 최소 신뢰도보다 낮으면 유효 음정 없음으로 판정합니다.
5. 최고 상관점의 양옆 값을 이용한 포물선 보간으로 주기를 세분화하고 `sample_rate / period`로 주파수를 구합니다.
6. 유효 주파수에는 지수형 평활을 적용하고, 유효 입력이 없을 때에는 평활 주파수를 빠르게 감쇠시킵니다.

### 3.2 세 상태 분류식

다음 기호를 사용합니다.

- `p`: 평활된 주파수
- `M`: 사용자가 조정하는 기준 주파수
- `g_min`: 최소 중립 간격
- `r`: 기준 주파수에 비례하는 중립 간격 비율
- `G = max(g_min, M × r)`: 실제 중립 간격

유효한 입력에 대한 분류는 다음과 같습니다.

```text
p <= M - G 이고 p >= 분석 최저 주파수  -> RETURN
p >= M + G                              -> ATTACK
그 밖의 유효 음정                       -> FLOAT
무음 또는 신뢰도 미달                    -> FLOAT
```

`RETURN`과 `ATTACK`은 같은 후보가 연속으로 확인된 뒤 확정합니다. `FLOAT`는 안전한 중립 상태로 즉시 전환하여 이전 공격 또는 귀환 명령이 불확실한 입력 때문에 유지되지 않게 합니다.

## 4. 상태별 운동

### `RETURN`

- 목표점은 화면 아래쪽의 홈 중심입니다.
- 현재 진행각에서 홈 방향으로 한 프레임에 허용된 최대 각도만큼 선회합니다.
- 속력은 홈과의 거리에 비례해 줄되 상한을 둡니다.
- 홈 중심 가까이에 도달하면 속도를 추가 감쇠합니다.

### `FLOAT`

- 직전 속도를 프레임 시간에 따라 감쇠합니다.
- 수평·수직 축에 서로 다른 저주파 진동을 조금 더해 정지 대신 완만한 부유 궤적을 만듭니다.
- 목표 선택과 적 격추 판정은 수행하지 않습니다.

### `ATTACK`

- 화살에서 가장 가까운 피격되지 않은 적을 자동 선택합니다.
- 현재 진행각에서 목표 방향으로 한 프레임에 허용된 최대 각도만큼 선회합니다.
- 목표가 있으면 일정한 공격 속력으로 이동합니다.
- 화살과 적의 거리가 충돌 반경보다 작을 때 적을 피격 상태로 바꿉니다. 기준 구현에서 적 격추는 `ATTACK` 상태에서만 가능합니다.

## 5. 게임 루프와 피드백

- 적은 화면의 위·왼쪽·오른쪽 바깥에서 생성되어 홈으로 직선 이동합니다.
- 적이 홈 반경에 닿으면 홈의 체력이 감소합니다. 체력이 0이면 게임이 종료됩니다.
- 공격 명령 중에는 선택된 적에 잠금 표시와 화살-목표 연결선을 그립니다.
- 피격된 적은 입자 효과와 함께 사라지고 새 적으로 보충됩니다.
- 화면에는 홈 체력, 점수, 격추 수, 평활 주파수, 현재 명령과 음정 신뢰도를 표시합니다.

## 6. HTML 기준 구현의 매개변수

아래 값은 기믹의 필수 법칙이 아니라 기준 파일을 구별하고 동일 동작을 재현하기 위한 스냅샷입니다.

| 항목 | 기준값 |
| --- | ---: |
| 기본 기준 주파수 `M` | 350 Hz |
| 기준 주파수 조절 범위 | 150–900 Hz, 10 Hz 간격 |
| 음정 분석 범위 | 120–900 Hz |
| RMS 최소값 | 0.003 |
| 자기상관 최소 신뢰도 | 0.46 |
| 중립 간격 `G` | `max(25 Hz, M × 0.075)` |
| 유효 음정 평활 | 이전값 0.62 + 새 값 0.38 |
| 무음 시 평활값 감쇠 계수 | 0.18 |
| 비중립 명령 확정 | 같은 후보 2회 연속 |
| 오디오 분석 버퍼 | 2,048 samples |
| 홈 반경 / 초기 체력 | 36 px / 10 |
| 동시 생존 적 상한 | 3 |
| 적 반경 | 20 px 이상 32 px 미만 |
| 공격 속력 / 최대 선회량 | 10 / `0.22 × dt` rad |
| 귀환 속력 | `min(10, 홈까지 거리 × 0.11)` |
| 귀환 최대 선회량 | `0.18 × dt` rad |
| 부유 속도 감쇠 | `0.92 ^ dt` |
| 화살 화면 경계 여백 | 20 px |
| 궤적 보관 길이 | 40 points |

시간 계수는 기준 구현에서 `dt = min((현재 프레임 시각 - 이전 프레임 시각) / 16, 3)`으로 계산합니다. 표의 속력은 이 정규화된 `dt`와 함께 사용되는 Canvas 좌표계 기준값입니다.

## 7. 상태 전이 의사코드

```text
samples = capture_microphone()
raw_pitch, confidence = estimate_pitch(samples)
smoothed_pitch = smooth_or_decay(raw_pitch)
command = classify(smoothed_pitch, confidence, adjustable_mark)

if command == RETURN:
    steer_persistent_arrow_toward(home)
elif command == ATTACK:
    target = nearest_live_enemy()
    steer_persistent_arrow_toward(target)
    allow_enemy_hit()
else:
    damp_velocity_and_float()

integrate_arrow_without_respawning_it()
move_enemies_toward(home)
resolve_hits_and_home_damage()
render_pitch_and_target_feedback()
```

## 8. Godot 후속 구현의 일치 기준

Godot Compatibility 포트는 이 기준 구현 이후의 별도 결과물입니다. 포트가 같은 기믹을 구현했다고 보기 위한 최소 동작 기준은 다음과 같습니다.

- 단일 화살의 위치와 속도가 세 상태 사이에서 연속적으로 유지될 것
- 조절 가능한 기준 주파수와 중립 구간을 사용해 세 명령을 분류할 것
- 불확실한 입력과 무음이 즉시 `FLOAT`로 귀결될 것
- `ATTACK`이 최근접 생존 적을 자동 선택하고 제한 선회로 추적할 것
- `RETURN`이 홈을 자동 추적하고 접근 시 감속할 것
- 음정 입력 없이도 상태 기계를 검증할 수 있는 디버그 입력 경로를 둘 것

엔진의 오디오·물리 단위 차이 때문에 수치를 조정할 수 있으나, 조정된 값은 Godot 버전의 별도 기록에 남깁니다.
