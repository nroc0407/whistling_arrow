## 5. Godot 포트와 버전 경계

Godot Compatibility 포트는 HTML 기준 구현을 바탕으로 진행하는 후속 구현입니다. 따라서 Godot 프로젝트 파일, GDScript, 장면과 내보내기 결과물은 위 `003046e` 기준 스냅샷에 포함되지 않습니다.

Godot 포트의 의미 있는 마일스톤마다 다음을 별도 항목으로 추가합니다.

1. 커밋 전체 해시와 태그
2. 핵심 프로젝트 파일의 SHA-256 매니페스트
3. 실행 엔진 버전과 렌더러
4. 마이크 입력 및 디버그 입력으로 세 상태가 동작하는 영상
5. HTML 기준과 달라진 매개변수 및 동작

이렇게 해야 원래 브라우저 구현과 엔진 포팅 결과물의 계보를 섞지 않고 설명할 수 있습니다.

첫 Godot 실행 가능 마일스톤을 [`godot-port-v1.md`](./godot-port-v1.md)에 별도로 기록했습니다. 핵심 소스 식별값은 [`godot-port-v1.sha256`](./godot-port-v1.sha256)에 있으며, 로컬 태그 `godot-port-v1`가 해당 포트 커밋을 가리킵니다. 이 태그 역시 원격에 푸시하기 전에는 외부 시점 증명이 아니라 로컬 계보 표식입니다.

## 6. 재검증 방법

PowerShell에서 기준 파일 해시를 다시 계산하려면 저장소 루트에서 다음 명령을 사용합니다.

```powershell
Get-FileHash -Algorithm SHA256 -LiteralPath .\whistling_arrow.html, .\ghs.docx
```

Git이 설치된 환경에서는 다음 명령으로 기준 이력을 확인할 수 있습니다.

```sh
git rev-parse HEAD
git log --reverse --format="%H %aI %s"
git status --short
```

파일 크기와 해시가 위 표와 다르면 내용이 변경된 것입니다. 정상적인 후속 변경이라면 기존 행을 덮어쓰지 말고 날짜와 커밋을 붙인 새 스냅샷 절을 추가합니다.
