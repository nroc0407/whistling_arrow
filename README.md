# Whistling Arrow

마이크로 감지한 휘파람 음정을 세 가지 명령으로 분류하여, 하나의 지속형 화살을 귀환·부유·자동추적 상태로 제어하는 2D 게임 프로토타입입니다.

## 현재 구현

- 기준 구현은 [`whistling_arrow.html`](./whistling_arrow.html)에 들어 있는 브라우저용 단일 파일 프로토타입입니다.
- [`ghs.docx`](./ghs.docx)는 기획 및 개발 기록입니다.
- [`godot/`](./godot/)에는 Godot 4.6.1 Compatibility 후속 포트가 있습니다. Godot 결과물은 HTML 기준 스냅샷에 소급하여 포함되지 않으며 별도 `godot-port-v1` 마일스톤과 해시로 기록합니다.

기준 Git 이력은 최초 커밋 `7eb0ec0`(2026-04-20)부터 기준 HEAD `003046e`(2026-06-15)까지입니다. 파일 식별값과 기록의 한계는 [창작·출처 기록](./docs/provenance.md)에 정리되어 있습니다.

## 핵심 조작

| 입력 | 명령 | 화살 동작 |
| --- | --- | --- |
| 기준보다 낮은 유효 음정 | `RETURN` | 홈을 향해 선회하고 접근하면서 감속 |
| 무음, 불확실한 입력, 기준 주변 음정 | `FLOAT` | 기존 속도를 감쇠하며 완만하게 부유 |
| 기준보다 높은 유효 음정 | `ATTACK` | 가장 가까운 살아 있는 적을 자동 선택하여 추적 |

플레이어가 화살의 좌표를 직접 지정하지 않는다는 점, 그리고 공격할 때마다 새 발사체를 만드는 대신 동일한 화살이 세 상태 사이를 계속 오간다는 점이 핵심입니다. 정확한 분류식과 기준 수치는 [기믹 명세](./docs/mechanic-spec.md)를 참고하세요.

## 브라우저에서 실행

마이크 API를 사용하므로 파일을 직접 여는 것보다 `localhost` 또는 HTTPS 환경에서 제공하는 편이 안전합니다. 예를 들어 저장소 루트에서 다음과 같이 정적 서버를 실행한 뒤 `http://localhost:8000/whistling_arrow.html`을 엽니다.

```sh
python -m http.server 8000
```

브라우저에서 마이크 사용을 허용해야 합니다. 구현은 에코 제거, 소음 억제, 자동 이득 조절을 끈 원시 입력을 요청하므로 장치와 주변 소음에 따라 기준 음정을 조절해야 할 수 있습니다.

## 권리와 기록의 의미

이 저장소의 이력과 해시는 특정 내용의 파일이 존재했다는 사실을 식별하고 변경 계보를 설명하기 위한 자료입니다. 이것만으로 추상적인 게임 아이디어에 대한 독점권, 특허성, 최초 창작자 지위 또는 법적 우선권이 자동으로 확정되지는 않습니다. 별도 라이선스가 명시되지 않은 원 저작물에는 일반적인 저작권이 유보됩니다. 자세한 내용은 [`RIGHTS.md`](./RIGHTS.md)를 확인하세요.
